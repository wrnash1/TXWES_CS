### 5. Quiz 4: Operations and Autoscaling (Question Bank)
**Question 1**
You want to ensure that your application can survive the failure of a single Compute Engine instance. If an instance crashes or becomes unresponsive, you want Google Cloud to automatically delete the broken instance and replace it with a fresh, identical instance. Which feature should you implement?
A) An Unmanaged Instance Group
B) A Managed Instance Group (MIG) with an autohealing health check
C) A Global HTTP(S) Load Balancer
D) Cloud Monitoring Alerting Policies
*   **Correct Answer:** B) A Managed Instance Group (MIG) with an autohealing health check
*   **Distractor Analysis:**
    *   *Why A is incorrect:* Unmanaged instance groups are just arbitrary collections of heterogeneous VMs. They do not support autoscaling or autohealing.
    *   *Why C is incorrect:* A load balancer distributes traffic away from a dead instance, but it does not have the authority to delete the dead VM and provision a new one. The MIG does that.
    *   *Why D is incorrect:* Alerting policies will email you that the server is down, but they will not automatically recreate the VM for you.

**Question 2**
Your company requires that all system and application logs generated by Compute Engine instances be retained for a minimum of 5 years to comply with financial regulations. By default, Cloud Logging only retains these logs for 30 days. How can you automate the long-term retention of these logs?
A) Create a Cloud Monitoring Uptime Check.
B) Take daily snapshots of all Compute Engine boot disks.
C) Create a Log Sink to export the logs to a Cloud Storage bucket using the Archive storage class.
D) Write a startup script to email the logs to the administrator daily.
*   **Correct Answer:** C) Create a Log Sink to export the logs to a Cloud Storage bucket using the Archive storage class.
*   **Distractor Analysis:**
    *   *Why A is incorrect:* Uptime checks only verify if an application is responding over the network; they have nothing to do with log retention.
    *   *Why B is incorrect:* While snapshots capture the disk state, relying on disk backups for granular, searchable log retention over 5 years is highly inefficient and expensive compared to native log exporting.
    *   *Why D is incorrect:* Emailing logs is not a secure, reliable, or scalable method for meeting 5-year enterprise compliance requirements. Log sinks natively automate the routing of logs to cheap, durable storage.
