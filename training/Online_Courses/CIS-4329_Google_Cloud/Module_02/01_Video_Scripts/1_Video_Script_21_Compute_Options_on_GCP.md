### 1. Video Script 2.1: Compute Options on GCP
**Estimated Duration:** 7 minutes
**Visual:** Instructor on camera.
**Audio:** "Welcome to Module 2. When you migrate to Google Cloud, your first major decision is determining where your application code will actually run. GCP offers a spectrum of compute options, from Infrastructure as a Service (IaaS) to pure Serverless."
**Visual:** A spectrum graphic from left (IaaS / More Control) to right (Serverless / Less Management). Placed on the spectrum: Compute Engine (Far Left), Google Kubernetes Engine (Middle Left), App Engine (Middle Right), Cloud Run / Cloud Functions (Far Right).
**[Alt-text: A horizontal spectrum labeled 'Compute Options'. Left side is 'More Control (IaaS)' containing Compute Engine. Middle is 'Containerized' containing GKE. Right side is 'Serverless (PaaS/FaaS)' containing App Engine and Cloud Run.]**
**Audio:** "If you need absolute control over the operating system, you use **Compute Engine**. It's just a raw virtual machine. If you are using Docker containers and need orchestration, you use **Google Kubernetes Engine (GKE)**. If you just want to upload your source code and let Google handle the scaling, provisioning, and web servers, you use **App Engine**. Finally, if you want to run stateless containers that scale to zero when not in use, you use **Cloud Run**. The ACE exam heavily tests your ability to pick the right compute product."
