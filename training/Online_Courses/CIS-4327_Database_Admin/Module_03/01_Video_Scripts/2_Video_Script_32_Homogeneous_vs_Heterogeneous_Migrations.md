### 2. Video Script 3.2: Homogeneous vs Heterogeneous Migrations
**Estimated Duration:** 5 minutes
**Visual:** Graphic showing two migration paths. Path 1: MySQL to MySQL. Path 2: Oracle to PostgreSQL.
**[Alt-text: Left side: An arrow pointing from a local MySQL icon to a Cloud SQL MySQL icon, labeled 'Homogeneous (Native Tools)'. Right side: An arrow pointing from an Oracle database icon to a Cloud SQL PostgreSQL icon, with a gear in the middle labeled 'Heterogeneous (Schema Conversion)'.]**
**Audio:** "Migrating from an on-premises PostgreSQL database to Cloud SQL PostgreSQL is called a Homogeneous migration. The engines are the same, the SQL dialect is the same, so DMS handles it natively and easily. But what if you want to escape expensive Oracle licensing and migrate to PostgreSQL? That is a Heterogeneous migration. The engines are completely different. For this, you must first use a Schema Conversion tool to rewrite the Oracle stored procedures and schemas into PostgreSQL dialect before you can migrate the data."
