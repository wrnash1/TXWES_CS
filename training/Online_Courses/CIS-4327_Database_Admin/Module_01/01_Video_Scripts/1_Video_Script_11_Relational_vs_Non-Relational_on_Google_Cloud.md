### 1. Video Script 1.1: Relational vs. Non-Relational on Google Cloud
**Estimated Duration:** 6 minutes
**Visual:** Instructor on camera.
**Audio:** "Welcome to Cloud Database Administration. Google Cloud offers a massive portfolio of database services, and the number one skill of a Cloud Database Engineer is simply choosing the right tool for the job."
**Visual:** A quadrant chart. X-axis: Relational to Non-Relational. Y-axis: Regional to Global scale. 
**[Alt-text: A quadrant chart plotting Google Cloud databases. Cloud SQL is in the Relational/Regional quadrant. Cloud Spanner is in Relational/Global. Firestore is Non-Relational/Global. Bigtable is Non-Relational/High-Throughput.]**
**Audio:** "If your application needs strict ACID compliance—meaning transactions must be perfect, like a banking app—you need a Relational database like Cloud SQL or Cloud Spanner. If your application handles massive amounts of unstructured data, like social media feeds or IoT sensor data, you need a Non-Relational, or NoSQL, database like Firestore or Bigtable."
