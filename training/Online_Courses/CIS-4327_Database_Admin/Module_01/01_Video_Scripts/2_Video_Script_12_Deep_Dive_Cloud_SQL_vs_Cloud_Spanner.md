### 2. Video Script 1.2: Deep Dive: Cloud SQL vs Cloud Spanner
**Estimated Duration:** 5 minutes
**Visual:** Graphic showing a single giant server (Vertical Scaling) vs. many small servers (Horizontal Scaling).
**[Alt-text: Left side shows a single server growing larger, labeled 'Vertical Scaling (Cloud SQL)'. Right side shows one server turning into three interconnected servers, labeled 'Horizontal Scaling (Cloud Spanner)'.]**
**Audio:** "Both Cloud SQL and Cloud Spanner are fully managed relational databases. So when do you use which? Cloud SQL is a lift-and-shift replacement for your local MySQL, PostgreSQL, or SQL Server. However, it scales vertically—you have to add more CPU and RAM to the single instance. Cloud Spanner is Google's globally distributed database. It scales horizontally across regions while magically maintaining strict consistency. If you are building a global supply chain app, you need Spanner."
